# システム仕様

本ドキュメントは、コードから自明でないシステム挙動を補足するための仕様書です。

## 自動アップデート

`electron-updater` (`autoUpdater`) を用いて GitHub Releases から差分配信を受け取る。

### 起動シーケンス

1. `app.whenReady` 直後に `initializeUpdater()` を呼び、`autoUpdater` のイベントハンドラを登録する。
2. メインウィンドウ生成後に `scheduleStartupCheck(mainWindow)` を呼ぶ。
   - `webContents` がまだロード中であれば `did-finish-load` を待ってから 3 秒後に `checkForUpdates()` を実行する。
   - すでにロード済みなら 3 秒の遅延だけを挟む。
   - 起動チェックは 1 度のみ実行する (内部フラグで二重起動を抑止)。

### 開発モードでは完全 no-op

`process.env.NODE_ENV === 'development'` または `--dev` 引数が付与されている場合、初期化・チェック・ダウンロード・インストール・スケジューリングはすべて早期 return する。`yarn dev` で GitHub への通信が一切発生しないことを保証する。

### `autoUpdater` 設定

| プロパティ | 値 | 理由 |
|---|---|---|
| `autoDownload` | `false` | ユーザー確認後にのみダウンロードを開始するため |
| `autoInstallOnAppQuit` | `false` | アプリ終了時の意図しない自動再起動を避けるため |
| `logger` | `console` | console-bridge を経由して DevTools に流すため |

### イベントとブロードキャスト方針

| イベント | 内部 state | renderer へのブロードキャスト |
|---|---|---|
| `checking-for-update` | `checking` | しない (UI 無音) |
| `update-available` | `available` | する |
| `update-not-available` | `not-available` | しない (UI 無音) |
| `download-progress` | `downloading` + `progress` | する |
| `update-downloaded` | `downloaded` | する。`autoInstallOnDownloaded` フラグが true なら 1.5 秒後に `quitAndInstall` を自動実行 |
| `error` | `idle` (リセット) | する。`console.error` のみで UI には通知しない。`autoInstallOnDownloaded` を false に戻す |

### 主要 API

- `downloadUpdate()`: `autoInstallOnDownloaded = true` をセットしてから `autoUpdater.downloadUpdate()` を呼ぶ。失敗時はフラグを戻し `console.error` のみ。
- `quitAndInstall()`: `setImmediate` で全ウィンドウを閉じてから `autoUpdater.quitAndInstall(false, true)` を呼ぶ。

### IPC チャンネル

| 定数 | チャンネル名 | 方向 |
|---|---|---|
| `UPDATER_GET_STATE` | `updater:getState` | renderer → main (invoke) |
| `UPDATER_CHECK` | `updater:check` | renderer → main (invoke) |
| `UPDATER_DOWNLOAD` | `updater:download` | renderer → main (invoke) |
| `UPDATER_QUIT_AND_INSTALL` | `updater:quitAndInstall` | renderer → main (invoke) |
| `UPDATER_STATE_CHANGED` | `updater:stateChanged` | main → renderer (push) |

### Renderer UI (`UpdateNotification`)

アプリのルート (常時マウントされる場所) に 1 つ配置する。

- `idle` / `checking` / `not-available` / `error` は何も描画しない。
- `available`: 「新しいバージョン v{{version}} が利用可能です。アップデートしますか？」を MUI Snackbar (右下、`autoHideDuration={null}`) で表示。「アップデート」「後で」の 2 ボタン。
- 「後で」を押すとそのセッション中は再表示しない (`dismissed`)。次回 `available` 状態に再遷移したときにリセット。
- 「アップデート」で `download()` を呼ぶ。以降の表示は:
  - `downloading`: 進捗バー (LinearProgress determinate + パーセント)
  - `downloaded`: 「アップデートを適用しています…」+ 不定形 LinearProgress
- 購読は `subscribe → getState` の順とし、初期状態の取りこぼしを防ぐ。

### electron-builder の publish 設定

`electron-builder.yml`:

```yaml
publish:
  provider: github
  owner: kakehashi-inc
  repo: app_backup_restore
  releaseType: draft
```

- `repo:` はリポジトリ名のみ (URL ではない)。
- `releaseType: draft` のため、各プラットフォームでの `release:*` 実行は同一バージョンの 1 つの GitHub Draft Release に成果物を集約する。GitHub UI で "Publish release" を押すまでユーザーには配信されない。
